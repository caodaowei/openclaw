## 🎯 对接机制

### 大哥 ↔ 小白（主对接人）

**小白是唯一与大哥直接对接的角色**

```
大哥
  │
  ▼
小白（项目经理）
  │
  ├─ 分析需求
  ├─ 分配任务给团队
  ├─ 跟踪进度
  ├─ 汇总结果
  │
  ▼
汇报大哥
```

**小白的职责**:
- 接收大哥所有指令
- 分析并拆解任务
- 分配给团队成员
- 跟踪执行进度
- 汇总结果汇报
- 处理异常情况

**其他成员不直接与大哥交互**，所有沟通通过小白中转。

## ⚙️ 工作流程

### 标准项目流程

```
阶段1: 需求分析
├─ 大哥提出需求
├─ 小白分析理解
├─ 小红协助细化（按需激活）
└─ 小白输出任务清单

阶段2: 技术方案
├─ 小白分配给小灰
├─ 小灰制定技术方案
├─ 小灰评估工作量
└─ 小白确认排期

阶段3: 开发执行
├─ 小灰分配给小蓝
├─ 小蓝执行开发
├─ 小灰代码审查
└─ 小蓝部署测试环境

阶段4: 测试验证
├─ 小白唤醒小黑
├─ 小黑执行测试
├─ 小黑反馈问题
├─ 小蓝修复问题
└─ 小黑确认通过

阶段5: 上线交付
├─ 小蓝部署生产环境
├─ 小青更新文档
├─ 小白汇总结果
└─ 小白汇报大哥
```

## 💾 资源管理

### 负载均衡策略

| 时段 | 活跃角色 | AITest01负载 | 腾讯负载 |
|------|---------|-------------|---------|
| 需求阶段 | 小白+小红 | 中 | 低 |
| 开发阶段 | 小灰+小蓝 | 低 | 高 |
| 测试阶段 | 小黑+小灰 | 中 | 中 |
| 部署阶段 | 小蓝+小青 | 低 | 高 |
| 日常运维 | 小白+小青 | 低 | 低 |

### 休眠机制

**休眠角色**:
- 不占用 CPU/内存
- 保留配置文件
- 快速唤醒能力

**唤醒触发**:
- 定时任务（小青：每天2次）
- 任务分配（小白按需唤醒）
- 阶段切换（自动唤醒对应角色）
