## 1. 项目概述

### 1.1 核心理念

**狐语 = 团队沟通的桥梁**

- 让团队沟通**更方便**
- 让沟通记录**更高效**
- 让飞书集成**更顺畅**
- 让后续总结**更轻松**

### 1.2 非核心理念

❌ 不追求复杂的项目管理  
❌ 不追求完美的功能覆盖  
❌ 不追求华丽的界面效果

### 1.3 一句话描述

> 一个让 OpenClaw 团队沟通更方便、记录更高效、能与飞书集成的轻量级沟通平台。

---

## 2. 核心功能（简化）

### 2.1 沟通记录 ⭐ 核心

**目标**：记录团队每次沟通，方便查阅和总结

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 快速记录 | 一键创建沟通记录，支持Markdown | P0 |
| 沟通类型 | 需求/任务/问题/汇报/其他 | P0 |
| 参与人员 | 选择参与沟通的团队成员 | P0 |
| 时间记录 | 自动记录沟通时间 | P0 |
| 标签分类 | 添加标签，方便筛选 | P1 |

**沟通记录字段**：
- 标题（一句话概括）
- 类型（需求/任务/问题/汇报/其他）
- 参与人（多选）
- 内容（Markdown格式）
- 时间（自动记录）
- 标签（可选）

### 2.2 飞书集成 ⭐ 核心

**目标**：重要沟通自动推送到飞书，大哥随时掌握

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 即时通知 | 重要沟通完成后，自动发飞书 | P0 |
| 每日汇总 | 每天定时发送当日沟通摘要 | P0 |
| 问题升级 | 技术问题升级时，即时通知 | P0 |
| 简洁格式 | 飞书消息简洁，重点突出 | P0 |

**飞书通知场景**：
1. **需求确认后** → 发送需求摘要给大哥
2. **任务完成后** → 发送完成通知给相关人员
3. **问题升级时** → 即时通知大哥
4. **每日晚8点** → 发送当日沟通汇总

### 2.3 沟通总结 ⭐ 核心

**目标**：自动总结沟通内容，方便回顾

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 关键词提取 | 自动提取沟通关键词 | P1 |
| 决策点标记 | 标记沟通中的关键决策 | P1 |
| 待办生成 | 从沟通中提取待办事项 | P1 |
| 周期总结 | 每周/每月自动生成总结报告 | P2 |

### 2.4 简单项目关联

**目标**：沟通与项目简单关联，不复杂

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 项目创建 | 简单创建项目，仅名称+描述 | P1 |
| 沟通关联 | 沟通记录归属某个项目 | P1 |
| 项目筛选 | 按项目查看沟通记录 | P1 |

**项目字段（极简）**：
- 项目名称
- 项目描述（可选）
- 创建时间

❌ 不需要：进度管理、任务分配、甘特图等复杂功能

---

## 3. 非功能需求

### 3.1 使用体验

- **快速**：3秒内创建沟通记录
- **简洁**：界面简洁，无多余功能
- **直观**：沟通记录一目了然

### 3.2 性能要求

- 页面加载 < 1秒
- 支持6人团队并发
- 数据本地存储，不上云

### 3.3 集成要求

- 飞书通知稳定送达
- 支持飞书消息模板

## 5. 技术方案（简化）

### 5.1 架构

```
┌─────────────────┐
│   狐语 FoxChat   │
│  （Web界面）     │
├─────────────────┤
│  沟通记录管理    │
│  飞书通知发送    │
│  简单总结生成    │
├─────────────────┤
│  本地JSON文件    │
└─────────────────┘
```

### 5.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML + JS + CSS | 简单页面，不复杂 |
| 后端 | Node.js + Express | 轻量级API |
| 数据 | JSON文件 | 本地存储，简单可靠 |
| 通知 | 飞书Webhook | 消息推送 |

### 5.3 页面设计（极简）

**首页**：
- 最近沟通记录列表
- 快速创建按钮
- 今日统计

**创建页**：
- 标题输入
- 类型选择（下拉）
- 参与人选择（多选）
- 内容编辑（Markdown）
- 保存按钮

**详情页**：
- 沟通内容展示
- 回复/评论（可选）
- 飞书分享按钮

**项目页**（简单）：
- 项目列表
- 按项目筛选沟通
