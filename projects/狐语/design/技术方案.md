## 2. 数据库设计

### 2.1 表结构

```sql
-- 团队表
CREATE TABLE teams (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(50) NOT NULL,
    role VARCHAR(50) NOT NULL,
    avatar_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 沟通记录表
CREATE TABLE communications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    team_id UUID REFERENCES teams(id),
    title VARCHAR(200) NOT NULL,
    type VARCHAR(50) NOT NULL, -- demand/task/issue/report/other
    participants UUID[] NOT NULL,
    content TEXT NOT NULL,
    tags VARCHAR(50)[],
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_comm_team ON communications(team_id);
CREATE INDEX idx_comm_type ON communications(type);
CREATE INDEX idx_comm_created ON communications(created_at);
```

## 3. API 设计

### 3.1 RESTful API

```
GET    /api/teams              # 获取团队列表
POST   /api/teams              # 创建团队
GET    /api/teams/:id          # 获取团队详情

GET    /api/communications     # 获取沟通列表
POST   /api/communications     # 创建沟通
GET    /api/communications/:id # 获取沟通详情
PUT    /api/communications/:id # 更新沟通
DELETE /api/communications/:id # 删除沟通

GET    /api/users              # 获取用户列表
GET    /api/users/:id          # 获取用户详情
```

### 3.2 请求/响应示例

**创建沟通**:
```json
POST /api/communications
{
  "title": "需求确认：性能监控面板",
  "type": "demand",
  "participants": ["user-uuid-1", "user-uuid-2"],
  "content": "需要开发一个性能监控面板...",
  "tags": ["需求", "性能"],
  "teamId": "team-uuid"
}

Response:
{
  "id": "comm-uuid",
  "title": "需求确认：性能监控面板",
  "type": "demand",
  "participants": [...],
  "content": "...",
  "createdAt": "2026-03-19T16:55:00Z"
}
```
