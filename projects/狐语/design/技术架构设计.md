## 2. 技术栈选择

### 2.1 前端框架 + AI 辅助

**选择：Vue 3 + TypeScript + Vite + Cursor**

| 对比 | Vue 3 | React |
|------|-------|-------|
| 学习成本 | 低 | 中 |
| 性能 | 高 | 高 |
| 生态 | 丰富 | 丰富 |
| 未来趋势 | 上升 | 稳定 |
| 团队熟悉度 | 易上手 | 需学习 |

**理由**：
- Vue 3 组合式API，代码更清晰
- Vite 构建快，开发体验好
- TypeScript 类型安全

**AI 辅助开发**：
- **Cursor**：AI 原生 IDE，代码生成、重构、解释
- **使用方式**：
  1. 用 Cursor 写基础代码
  2. AI 生成组件模板
  3. 人调整优化
  4. AI 审查代码质量

### 2.2 后端框架

**选择：Node.js + Express + TypeScript**

**理由**：
- 与前端技术栈统一（JavaScript）
- Express 简单稳定，扩展性好
- TypeScript 类型安全，维护方便
- 生态丰富，组件多

### 2.3 数据存储

**选择：PostgreSQL**

```
优点：
- 企业级数据库，稳定可靠
- 支持复杂查询、JSON字段
- 后续项目可复用
- 生态丰富，工具多

部署：
- 开发：本地 Docker
- 生产：Docker 或云服务
```

**数据模型设计（PostgreSQL）**：

```typescript
// 沟通记录表
interface Communication {
  id: UUID;             // UUID，主键
  title: string;        // 标题
  type: CommType;       // 类型（enum）
  participants: UUID[]; // 参与人（外键数组）
  content: string;      // 内容（Markdown）
  tags: string[];       // 标签（数组）
  projectId?: UUID;     // 关联项目（外键，可选）
  createdAt: Timestamp; // 创建时间
  updatedAt: Timestamp; // 更新时间
  
  // 扩展字段（PostgreSQL JSONB）
  metadata?: JSONB;     // 预留扩展
  aiSummary?: string;   // AI 自动总结（v2.0）
  decisions?: string[]; // 决策点（v2.0）
  todos?: Todo[];       // 待办事项（v2.0）
}

// 项目表（极简）
interface Project {
  id: UUID;
  name: string;
  description?: string;
  createdAt: Timestamp;
  
  // 扩展字段
  metadata?: JSONB;
}

// 用户表（团队成员）
interface User {
  id: UUID;
  name: string;         // 显示名称
  role: UserRole;       // 角色（enum）
  avatar?: string;      // 头像（可选）
  
  // 扩展字段
  metadata?: JSONB;
}

// 数据库特性使用
// - UUID：分布式支持
// - JSONB：灵活扩展字段
// - Array：标签、参与人等
// - Enum：类型约束
```

### 2.4 API 设计（RESTful + 预留GraphQL）

**当前：RESTful API**

```
GET    /api/communications          # 获取沟通列表
POST   /api/communications          # 创建沟通
GET    /api/communications/:id      # 获取沟通详情
PUT    /api/communications/:id      # 更新沟通
DELETE /api/communications/:id      # 删除沟通

GET    /api/projects                # 获取项目列表
POST   /api/projects                # 创建项目

GET    /api/users                   # 获取用户列表
```

**未来扩展**：
- 可添加 GraphQL 接口
- 支持更灵活的数据查询
- 前后端解耦更彻底

## 5. 部署架构

### 5.1 部署方案

**Docker Compose 部署（推荐）**：

```yaml
# docker-compose.yml
version: '3.8'
services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@postgres:5432/foxchat
      - FEISHU_WEBHOOK_URL=${FEISHU_WEBHOOK_URL}
    depends_on:
      - postgres

  postgres:
    image: postgres:15-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_USER=foxchat
      - POSTGRES_PASSWORD=${DB_PASSWORD}
      - POSTGRES_DB=foxchat

volumes:
  postgres_data:
```

**部署环境**：
- **开发**：本地 Docker Compose
- **生产**：AITest01 + Docker

## 6. 数据迁移路径

### 6.1 数据库管理

- **开发**：pgAdmin 或 DBeaver

**备份策略**：
```bash
# 每天自动备份
```
