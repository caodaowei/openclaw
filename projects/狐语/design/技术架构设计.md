# 狐语 FoxChat 技术架构设计

> 简洁现在，扩展未来

---

## 1. 架构原则

### 1.1 简洁现在

- 当前仅支持6人团队
- 功能聚焦沟通记录
- 界面简洁够用

### 1.2 扩展未来

- 架构支持未来扩展
- 数据模型预留扩展字段
- 技术栈主流可持续

---

## 2. 技术栈选择

### 2.1 前端框架

**选择：Vue 3 + Vite**

| 对比 | Vue 3 | React |
|------|-------|-------|
| 学习成本 | 低 | 中 |
| 性能 | 高 | 高 |
| 生态 | 丰富 | 丰富 |
| 未来趋势 | 上升 | 稳定 |
| 团队熟悉度 | 易上手 | 需学习 |

**理由**：
- Vue 3 组合式API，代码更清晰
- Vite 构建快，开发体验好
- 轻量级，适合小项目
- 未来可平滑升级

### 2.2 后端框架

**选择：Node.js + Express + TypeScript**

**理由**：
- 与前端技术栈统一（JavaScript）
- Express 简单稳定，扩展性好
- TypeScript 类型安全，维护方便
- 生态丰富，组件多

### 2.3 数据存储

**当前：SQLite（本地文件）**

```
优点：
- 零配置，即开即用
- 单文件，备份简单
- 支持SQL，查询方便
- 性能足够（6人团队）

未来扩展：
- 可无缝迁移到 PostgreSQL
- 数据导出导入方便
```

**数据模型设计（预留扩展）**：

```typescript
// 沟通记录表
interface Communication {
  id: string;           // UUID，预留分布式
  title: string;        // 标题

  participants: string[]; // 参与人
  content: string;      // 内容（Markdown）

  updatedAt: Date;      // 更新时间
  
  // 扩展字段
  metadata?: Record<string, any>;  // 预留扩展

// 项目表（极简）
interface Project {
  id: string;
  name: string;
  description?: string;
  createdAt: Date;
  
  // 扩展字段
  metadata?: Record<string, any>;

// 用户表（团队成员）
interface User {
  id: string;

### 2.4 API 设计（RESTful + 预留GraphQL）

**当前：RESTful API**

```
GET    /api/communications          # 获取沟通列表
POST   /api/communications          # 创建沟通
GET    /api/communications/:id      # 获取沟通详情
PUT    /api/communications/:id      # 更新沟通
DELETE /api/communications/:id      # 删除沟通

GET    /api/projects                # 获取项目列表
POST   /api/projects                # 创建项目

GET    /api/users                   # 获取用户列表
```

**未来扩展**：
- 可添加 GraphQL 接口
- 支持更灵活的数据查询
- 前后端解耦更彻底

- 组件化设计，新增功能只需添加组件

- 分层架构（Controller -> Service -> Model）
- 中间件机制，易于添加功能

## 6. 数据迁移路径

### 6.1 当前 → 未来

```
SQLite → PostgreSQL

步骤：
1. 导出 SQLite 数据为 SQL
2. 转换 SQL 语法（如有差异）

4. 更新后端数据库连接配置
5. 测试验证
