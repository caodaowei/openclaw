## 2. 技术栈选择

### 2.1 前端框架 + AI 辅助

**选择：Vue 3 + TypeScript + Vite + Cursor**

| 对比 | Vue 3 | React |
|------|-------|-------|
| 学习成本 | 低 | 中 |
| 性能 | 高 | 高 |
| 生态 | 丰富 | 丰富 |
| 未来趋势 | 上升 | 稳定 |
| 团队熟悉度 | 易上手 | 需学习 |

**理由**：
- Vue 3 组合式API，代码更清晰
- Vite 构建快，开发体验好
- TypeScript 类型安全

**AI 辅助开发**：
- **Cursor**：AI 原生 IDE，代码生成、重构、解释
- **使用方式**：
  1. 用 Cursor 写基础代码
  2. AI 生成组件模板
  3. 人调整优化
  4. AI 审查代码质量

### 2.2 后端框架

**选择：Node.js + Express + TypeScript**

**理由**：
- 与前端技术栈统一（JavaScript）
- Express 简单稳定，扩展性好
- TypeScript 类型安全，维护方便
- 生态丰富，组件多

### 2.3 数据存储（可替换架构）

**当前：PostgreSQL**

```
优点：
- 企业级数据库，稳定可靠
- 支持复杂查询、JSON字段
- 后续项目可复用
- 生态丰富，工具多
```

**数据模型设计（数据库无关）**：

```typescript
// 沟通记录表（通用类型）
interface Communication {
  id: string;           //  string ID（UUID 或 ObjectId）
  title: string;        // 标题
  type: CommType;       // 类型（string enum）
  participants: string[]; // 参与人 ID 数组
  content: string;      // 内容（Markdown）
  tags: string[];       // 标签（数组）
  projectId?: string;   // 关联项目 ID（可选）
  createdAt: string;    // ISO 8601 时间字符串
  updatedAt: string;    // ISO 8601 时间字符串
  
  // 扩展字段（JSON 字符串，各数据库通用）
  metadata?: string;    // JSON 字符串，预留扩展
}

// 项目表（极简）
interface Project {
  id: string;
  name: string;
  description?: string;
  createdAt: string;
  
  // 扩展字段
  metadata?: string;    // JSON 字符串
}

// 用户表（团队成员）
interface User {
  id: string;
  name: string;         // 显示名称
  role: UserRole;       // 角色（string enum）
  avatar?: string;      // 头像（可选）
  
  // 扩展字段
  metadata?: string;    // JSON 字符串
}

// 数据库适配
// - 使用通用类型，不依赖特定数据库特性
// - 时间用 ISO 8601 字符串，各数据库都支持
// - 扩展字段用 JSON 字符串，兼容性好
// - 具体实现由 Repository 处理
```

## 5. 部署架构

### 5.1 部署方案

**Docker Compose 部署（推荐）**：

```yaml
# docker-compose.yml
version: '3.8'
services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@postgres:5432/foxchat
      - FEISHU_WEBHOOK_URL=${FEISHU_WEBHOOK_URL}
    depends_on:
      - postgres

  postgres:
    image: postgres:15-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_USER=foxchat
      - POSTGRES_PASSWORD=${DB_PASSWORD}
      - POSTGRES_DB=foxchat

volumes:
  postgres_data:
```

**部署环境**：
- **开发**：本地 Docker Compose
- **生产**：AITest01 + Docker

## 6. 数据迁移路径

### 6.1 数据库管理（可替换）

**当前使用 PostgreSQL，未来可无缝切换**

**切换步骤**：
```
1. 实现新数据库的 Repository 类
2. 数据导出（通用 JSON 格式）
3. 数据导入（新数据库）
4. 修改配置：database.type
5. 重启服务
```

**数据库管理工具**：
- **开发**：DBeaver（支持多种数据库）
- **备份**：通用 JSON 导出
- **监控**：应用层监控

**备份策略（通用）**：
```bash
# 每天自动备份为 JSON
0 2 * * * node scripts/backup.js > /backup/foxchat_$(date +\%Y\%m\%d).json
```

**支持的数据库（未来扩展）**：
